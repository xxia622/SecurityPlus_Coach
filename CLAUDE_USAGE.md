# Claude 使用手册
**如何把 Claude 的效率最大化**

---

## 核心原则：给越多上下文，越有用

差的问法：`"Python怎么学"`
好的问法：`"我要用 Python 写安全工具，现在零基础，给我一个4周实战计划"`

---

## 最高效的使用方式

### 1. 把任务直接扔给我，不只是问问题
我可以直接读文件、跑命令、写代码、推 GitHub。

- "帮我写一个 Python 脚本扫描开放端口"
- "我的代码报错了" → 直接粘代码过来
- "帮我把这个 PDF 总结成笔记"

### 2. Security+ 备考
每天学完后来问：
- "用场景解释给我听什么是 MFA"
- "出10道关于 Domain 2 的模拟题"
- "我答错了这道题，帮我讲清楚"

### 3. 课程作业
把题目要求发给我，我帮你理解题意、debug、检查逻辑。
> 让我帮你理解和 debug 是正常的，直接替你写交上去有学术诚信风险。

### 4. 职业规划执行
- "帮我写一份针对网络安全实习的简历"
- "帮我准备这个岗位的面试题"
- "帮我看看这个 CTF 题怎么解"

## /clear 的使用规则

| 情况 | 要不要 /clear |
|------|--------------|
| 换了完全不相关的新任务 | ✅ 要 |
| 上一个任务已经完成 | ✅ 要 |
| 对话太长、回复变慢 | ✅ 要 |
| 同一个任务还没做完 | ❌ 不要 |
| 话题相关联 | ❌ 不要 |

**重要：** `/clear` 只清对话上下文，不影响记忆文件。个人背景、考试日期、职业规划这些已存入记忆的内容 `/clear` 后依然保留。
